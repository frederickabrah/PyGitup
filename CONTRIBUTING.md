# Contributing to PyGitUp

First off, thank you for considering contributing to PyGitUp! It's people like you that make the open-source community such an amazing place. We welcome any and all contributions, from bug reports to feature requests and code patches.

## How Can I Contribute?

### Reporting Bugs

If you find a bug, please open an issue on our GitHub repository. When you report a bug, please include:

*   Your operating system and Python version.
*   A detailed description of the bug and the steps to reproduce it.
*   The full traceback of any errors you encounter.
*   The expected behavior and what happened instead.

This information will help us to quickly identify and fix the problem.

### Suggesting Enhancements

If you have an idea for a new feature or an improvement to an existing one, please open an issue on our GitHub repository. We would love to hear your ideas!

### Submitting Code Changes

If you would like to contribute code to fix a bug or add a new feature, please follow these steps:

1.  **Fork the repository:** Create your own fork of the PyGitUp repository on GitHub.
2.  **Create a new branch:** Create a new branch for your changes. Please use a descriptive branch name (e.g., `fix-issue-123`, `add-feature-x`).
3.  **Make your changes:** Make your changes to the code and add or update tests as appropriate.
4.  **Run the tests:** Make sure that all tests pass by running `python -m unittest discover tests`.
5.  **Commit your changes:** Commit your changes with a clear and descriptive commit message.
6.  **Push your changes:** Push your changes to your forked repository.
7.  **Open a pull request:** Open a pull request from your branch to the `main` branch of the PyGitUp repository. Please provide a clear description of your changes in the pull request.

## Code Style

We follow the [PEP 8](https://www.python.org/dev/peps/pep-0008/) style guide for Python code. Please make sure that your code adheres to these guidelines.

## Code of Conduct

Please note that this project is released with a Contributor Code of Conduct. By participating in this project you agree to abide by its terms. Please see `CODE_OF_CONDUCT.md` for more details.

Thank you again for your interest in contributing to PyGitUp!
